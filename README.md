# Моя смена

«Моя смена» — локальный трекер рабочего времени, перерывов, заработка и личной статистики. Приложение работает без регистрации и облачного сервера: рабочие записи хранятся в браузере на этом компьютере.

> Это личный локальный трекер. Он не является официальным сервисом Яндекса и не заменяет внутреннюю систему учёта рабочего времени.

## Что понадобится

- Windows 10 или 11.
- Node.js **20.19 или новее** либо **22.12 или новее**. Рекомендуется актуальная LTS-версия с [официального сайта Node.js](https://nodejs.org/).
- npm — устанавливается вместе с Node.js.
- Актуальная стабильная версия Chrome, Edge или Firefox для основного интерфейса.

После установки Node.js откройте новую командную строку и проверьте:

```bat
node --version
npm --version
```

## Быстрый запуск в Windows

1. Дважды щёлкните `start.bat`.
2. При первом запуске скрипт установит зависимости.
3. После готовности сервера автоматически откроется `http://127.0.0.1:4173/`.
4. Для остановки локального сервера запустите `stop.bat`.

`start.bat` собирает production-версию с PWA/offline-поддержкой и запускает её только локально на `127.0.0.1`. Его PID записывается в `.moya-smena.pid`. `stop.bat` проверяет, что PID действительно принадлежит Vite этого проекта, и не завершает другие процессы Node. Диагностический вывод сохраняется в `.moya-smena.stdout.log` и `.moya-smena.stderr.log`.

Если порт `4173` занят другой программой, сервер не запустится: освободите порт и повторите запуск.

## Ручной запуск

Откройте терминал в папке проекта:

```bat
npm ci
npm run dev
```

Затем откройте `http://127.0.0.1:4173/`. Для остановки процесса, запущенного в текущем терминале, нажмите `Ctrl+C`.

Не меняйте адрес на `localhost`, если уже начали работать через `127.0.0.1`: для браузера это разные источники, поэтому у них разные хранилища IndexedDB и отдельные наборы данных.

## Расчёт заработка по БО

Количество БО вводится вручную после завершения смены или позднее через календарь. Приложение не получает БО из рабочих систем.

Фиксированный курс: **1 БО = 0,80 ₽**.

```text
Начислено за БО = количество БО × 0,80 ₽
Итоговый заработок = начислено за БО + премия/доплата − удержание
```

Например, **350 БО = 280 ₽**. Если добавить премию 100 ₽ и удержание 50 ₽, итоговый заработок составит 330 ₽.

Если БО ещё неизвестны, оставьте поле пустым и заполните его позже в календаре. Пустое значение не считается нулём. Старые записи, ранее введённые напрямую в рублях, сохраняются без автоматической подмены на БО.

## Проверки и production-сборка

```bat
npm run typecheck
npm run lint
npm test
npm run build
```

Готовая production-сборка появится в папке `dist`. Для локальной проверки сборки сначала остановите dev-сервер, затем выполните:

```bat
npm run preview
```

Preview откроется на том же адресе: `http://127.0.0.1:4173/`.

Для интерактивного режима тестов можно использовать:

```bat
npm run test:watch
```

## Установка как PWA

1. Выполните `npm run build`.
2. Запустите `npm run preview`.
3. Откройте `http://127.0.0.1:4173/` в браузере.
4. Дождитесь полной загрузки страницы и регистрации Service Worker.
5. В браузере, поддерживающем установку desktop-PWA, используйте значок установки в адресной строке или пункт меню «Установить приложение». Если такого пункта нет, продолжайте пользоваться локальным сайтом: доступность установки определяется самим браузером.

Установленное PWA запускается в отдельном окне. После первой полной загрузки production-версии основные функции доступны без интернета. При появлении уведомления о новой версии подтвердите обновление и перезапустите приложение.

Режим приватного просмотра для постоянного хранения не подходит: браузер может удалить его данные после закрытия.

## Где хранятся данные

Смены, перерывы, настройки, количество БО, курс конкретной смены, рассчитанный заработок и планы хранятся локально в **IndexedDB** профиля браузера для источника `http://127.0.0.1:4173`.

- Данные не отправляются на удалённый сервер.
- Разные браузеры и разные профили браузера имеют отдельные базы.
- Очистка данных сайта или удаление профиля браузера удалит записи.
- Удаление папки проекта не является резервным копированием базы браузера.

Перед очисткой браузера обязательно сохраните JSON-резервную копию.

## Резервная копия, импорт и CSV

Откройте **«Настройки» → «Данные»**.

- **Экспорт JSON** сохраняет полную резервную копию смен, планов и настроек.
- **Экспорт CSV** создаёт таблицу смен, БО, курса и заработка для просмотра в табличном редакторе.
- **Импорт JSON** сначала проверяет структуру файла и показывает предварительный просмотр.
- Режим **«Объединить»** добавляет совместимые данные к существующим.
- Режим **«Заменить»** полностью заменяет локальную базу после отдельного подтверждения.

Храните JSON-файл в надёжном месте. Не редактируйте его вручную без необходимости: повреждённый или несовместимый файл будет отклонён.

Чтобы восстановить данные на другом компьютере или в другом браузере, откройте приложение там и импортируйте ранее сохранённый JSON.

## Полное удаление локальных данных

1. При необходимости сначала экспортируйте JSON.
2. Откройте **«Настройки» → «Данные»**.
3. Нажмите кнопку удаления всех локальных данных и подтвердите действие.

Также данные можно удалить через настройки браузера для сайта `127.0.0.1:4173`. Это действие необратимо без резервной копии. Удаление PWA само по себе не во всех браузерах гарантирует удаление IndexedDB, поэтому проверьте данные сайта отдельно.

## Таймер поверх всех окон

Во время активной смены нажмите **«Показать поверх всех окон»**. Открытие должно происходить непосредственно по клику пользователя — браузеры запрещают вызывать этот API автоматически.

Приложение определяет поддержку только программно:

```ts
if ('documentPictureInPicture' in window) {
  // настоящее окно Document Picture-in-Picture
}
```

Если браузер предоставляет Document Picture-in-Picture, откроется настоящее компактное окно, которое может оставаться поверх других программ. Базовый API поддерживается в настольных Google Chrome 116+, Microsoft Edge 116+ и Mozilla Firefox 151+. На мобильных версиях этих браузеров режим недоступен. Даже при подходящей версии приложение всегда проверяет API непосредственно перед открытием: политики или настройки браузера могут его отключить. Определение по `userAgent` не используется.

### Chrome

1. Откройте приложение, начните смену и нажмите «Показать поверх всех окон».
2. В настольном Chrome 116+ при доступном `documentPictureInPicture` откроется настоящее окно поверх программ.
3. Если API недоступен или запрещён политикой браузера, используйте предложенное компактное fallback-окно либо основной таймер.

### Edge

1. Откройте приложение, начните смену и нажмите «Показать поверх всех окон».
2. В настольном Edge 116+ при доступном `documentPictureInPicture` откроется настоящее окно поверх программ.
3. При запрете API или его отсутствии следуйте сообщению приложения и используйте fallback.

### Firefox

1. Откройте приложение, начните смену и нажмите «Показать поверх всех окон».
2. Настоящий Document Picture-in-Picture доступен в настольном Firefox 151+.
3. Приложение всё равно проверит наличие `documentPictureInPicture` в реально установленной версии. Если API отсутствует, сразу откроется запасной компактный режим. Если браузер сначала отклонил PiP и одновременно заблокировал popup, повторное нажатие резервирует отдельный пользовательский жест для обычного окна.

Если API отсутствует или запрос запрещён, приложение использует отдельное компактное окно. Это запасной режим: обычное всплывающее окно **не гарантирует** положение поверх других программ. Если первая попытка была заблокирована после отказа PiP, нажмите кнопку ещё раз; если окно всё равно не открылось, разрешите всплывающие окна для `127.0.0.1`.

Настоящий режим поверх окон работает только пока основной сайт или установленное PWA остаётся открытым. Мини-окно закрывается вместе с браузером и может закрыться при закрытии исходной вкладки или PWA. Приложение не обещает работу таймера или системных уведомлений после полного закрытия браузера.

Справка по совместимости: [Chrome Developers](https://developer.chrome.com/docs/web-platform/document-picture-in-picture), [Firefox 151](https://developer.mozilla.org/en-US/docs/Mozilla/Firefox/Releases/151), [MDN requestWindow()](https://developer.mozilla.org/en-US/docs/Web/API/DocumentPictureInPicture/requestWindow).

Если кнопка недоступна:

1. Убедитесь, что смена начата.
2. Обновите браузер до актуальной стабильной версии.
3. Проверьте разрешения и корпоративные политики браузера.
4. Нажмите кнопку повторно именно в основном окне приложения.
5. Используйте основной таймер или предложенный fallback, если Document Picture-in-Picture не поддерживается.

## Уведомления и конфиденциальность

Разрешение на системные уведомления запрашивается только после действия пользователя. Если уведомления запрещены, приложение продолжает показывать напоминания внутри открытого сайта. Доставка системных уведомлений после полного закрытия браузера не гарантируется.

Не сохраняйте в заметках персональные данные клиентов и содержимое обращений. Приложение не предназначено для переписок, имён, телефонов, адресов, номеров заказов, снимков экрана или другой клиентской информации.
